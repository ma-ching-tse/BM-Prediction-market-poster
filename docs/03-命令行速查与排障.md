# 命令行速查与排障

## 1. 基础命令

安装依赖（首次或依赖变更后）：

```bash
npm install
```

查看帮助：

```bash
node generate.js --help
```

生成 Classic（默认模板）：

```bash
npm run generate
# 等价于 node generate.js
# 等价于 node generate.js --template classic
```

生成 Comprehensive：

```bash
npm run generate:comprehensive
# 等价于 node generate.js --template comprehensive
```

## 2. 模板参数别名

`classic` 支持别名：

1. `classic`
2. `default`
3. `标准`
4. `默认`

`comprehensive` 支持别名：

1. `comprehensive`
2. `event`
3. `综合事件`
4. `综合事件模版`

## 3. 产物定位

查最近产物：

```bash
ls -lt output/NBA
ls -lt output/综合事件
```

进入某天目录：

```bash
ls -lh output/NBA/<YYYYMMDD>
ls -lh output/综合事件/<YYYYMMDD>
```

## 4. 快速自检（运行前）

```bash
# 1) 配置文件是否存在
ls -l lark.config.json

# 2) 背景图是否齐全（决定输出语种）
ls backgrounds

# 3) 模板文案配置是否存在
ls -l poster.copy.json

# 4) 球队映射是否存在（classic 需要）
ls -l teams.csv
```

## 5. 常见报错与处理

1. `未找到配置文件：lark.config.json`
- 处理：补齐 `lark.config.json`，确认 `appId` / `appSecret` / `spreadsheetToken` 等字段。

2. `Lark 表格缺少字段：date/home_team/away_team`
- 处理：检查 Lark 表头是否和脚本要求一致。

3. `胜率校验失败（home_win + away_win 必须等于 100）`
- 处理：修正 Lark 里的胜率，保证每场加总 100。

4. `找不到球队 Logo`
- 处理：检查 `teams.csv` 的 `logo` 值与 `NBA_icon/*.png` 文件名是否完全一致。

5. `backgrounds/ 目录下没有找到背景图`
- 处理：在 `backgrounds/` 放入至少 1 张背景图（如 `zh-CN.png`）。

6. `Polymarket outcome 映射失败`
- 处理：在 Lark 补 `polymarket_slug` 或 `home_outcome` / `away_outcome` 做人工指定。

## 6. 运营推荐执行顺序

1. 在 Lark 改数据
2. 本地执行对应模板命令
3. 检查输出目录和 ZIP
4. 抽查 2-3 张多语言图片
5. 交付 ZIP 给运营发布
