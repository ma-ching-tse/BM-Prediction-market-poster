# 命令行速查与排障

## 1. 基础命令

安装依赖（首次或依赖变更后）：

```bash
npm install
```

检查 `ffmpeg` 是否可用：

```bash
ffmpeg -version
```

若未安装（macOS）：

```bash
brew install ffmpeg
```

查看帮助：

```bash
node generate.js --help
```

生成 Classic（默认模板）：

```bash
npm run generate
# 等价于 node generate.js
# 等价于 node generate.js --template classic
```

生成 Comprehensive：

```bash
npm run generate:comprehensive
# 等价于 node generate.js --template comprehensive
```

生成 World Cup：

```bash
npm run generate:worldcup
# 等价于 node generate.js --template worldcup
```

生成 Football：

```bash
npm run generate:football
# 等价于 node generate.js --template football
```

生成 F1 车队：

```bash
npm run generate:f1
# 等价于 node generate.js --template f1
```

生成 F1 车手：

```bash
npm run generate:f1driver
# 等价于 node generate.js --template f1driver
```

## 4. 快速自检（运行前）

```bash
# 1) 配置文件是否存在
ls -l lark.config.json

# 2) 背景图是否齐全（决定输出语种）
ls backgrounds-NBA
ls "backgrounds- football"
ls backgrounds-F1

# 3) 模板文案配置是否存在
ls -l poster.copy.json

# 4) 球队映射是否存在（模板按需）
ls -l teams.csv
ls -l football_teams.csv
ls -l teams_f1.csv
ls -l drivers_f1.csv

# 5) ffmpeg 是否可用（PNG 压缩依赖）
ffmpeg -version
```

## 5. 常见报错与处理

1. `未找到配置文件：lark.config.json`
- 处理：补齐 `lark.config.json`，确认 `appId` / `appSecret` / `spreadsheetToken` 等字段。

2. `Lark 表格缺少字段：date/home_team/away_team`
- 处理：检查 Lark 表头是否和脚本要求一致。

3. `胜率校验失败（home_win + away_win 必须等于 100）`
- 处理：修正 Lark 里的胜率，保证每场加总 100。

4. `找不到球队 Logo`
- 处理：检查 `teams.csv` 的 `logo` 值与 `NBA_icon/*.png` 文件名是否完全一致。

5. `背景目录下没有找到背景图`
- 处理：在对应目录放入至少 1 张背景图（如 `zh-CN.png`），例如 `backgrounds-NBA/`、`backgrounds-F1/`、`backgrounds- football/`。

6. `Polymarket outcome 映射失败`
- 处理：检查 `match1_link ~ match3_link` 是否是对应比赛的有效 Polymarket 场次链接。

7. `ffmpeg 压缩失败` 或 `command not found: ffmpeg`
- 处理：先执行 `brew install ffmpeg`，再重跑命令。

## 6. 运营推荐执行顺序

1. 在 Lark 改数据
2. 本地执行对应模板命令
3. 检查输出目录和 ZIP
4. 抽查 2-3 张多语言图片
5. 交付 ZIP 给运营发布
