# 预测市场海报模板更新总览（交付运营）

## 1. 当前可用模板（以代码为准）

当前已统一登记并可直接调用的模板共 8 个：

1. `classic`（NBA 赛事）
2. `comprehensive`（综合事件）
3. `worldcup`（世界杯）
4. `football`（足球赛事）
5. `coinprice`（币价预测）
6. `global`（全球预测市场）
7. `f1`（F1 车队）
8. `f1driver`（F1 车手）

对应关系：

- `classic` -> 模板文件 `poster.html` -> 输出目录 `output/NBA/`
- `comprehensive` -> 模板文件 `poster.comprehensive-event.html` -> 输出目录 `output/综合事件/`
- `worldcup` -> 模板文件 `poster.world-cup.html` -> 输出目录 `output/世界杯/`
- `football` -> 模板文件 `poster.football-soccer.html` -> 输出目录 `output/足球赛事/`
- `coinprice` -> 模板文件 `poster.coin-price.html` -> 输出目录 `output/币价预测/`
- `global` -> 模板文件 `poster.global-prediction-market.html` -> 输出目录 `output/全球预测市场/`
- `f1` -> 模板文件 `poster.f1.html` -> 输出目录 `output/F1 车队/`
- `f1driver` -> 模板文件 `poster.f1driver.html` -> 输出目录 `output/F1 Driver/`

## 2. 统一更新入口

所有模板统一命令入口：

```bash
node generate.js --template <模板名>
```

或使用 npm 脚本：

```bash
npm run generate
npm run generate:comprehensive
npm run generate:worldcup
npm run generate:football
npm run generate:coinprice
npm run generate:global
npm run generate:f1
npm run generate:f1driver
```

## 3. 模板更新方式（运营视角）

每个模板都可以从 4 个层面更新：

1. 数据源更新（Lark 表格）
2. 文案/字体参数更新（`poster.copy.json`）
3. 视觉模板更新（对应 `poster*.html` 文件）
4. 素材更新（`backgrounds/`、`assets/`、`NBA_icon/`）

## 4. 通用前置条件

1. Node 环境可用，且已安装依赖：`npm install`
2. 本机有 Chrome：`/Applications/Google Chrome.app/Contents/MacOS/Google Chrome`
3. `lark.config.json` 配置正确（此文件被 `.gitignore` 忽略，不入库）

## 5. 产物与交付

执行后会自动：

1. 按语种背景图生成多语言 JPG（每张压缩到约 300KB 内）
2. 产出同名 ZIP 包
3. 输出到 `output/<模板目录>/<YYYYMMDD>/`

例如：

- `output/NBA/20260416/`
- `output/综合事件/20260416/`

## 6. 推荐阅读顺序

1. 先看 `docs/03-命令行速查与排障.md`
2. 做 NBA 赛事海报看 `docs/01-Classic模板更新手册.md`
3. 做综合事件海报看 `docs/02-Comprehensive模板更新手册.md`
