# 运营同学海报生成手册（8 个模板）

在飞书表格里填好数据，启动本地服务，打开网页点一下即可出图。

---

## 第一步：启动服务（每次开机后做一次就够）

> 这一步需要用到「终端」（Terminal），以下是完整操作方法，做过一次就熟了。

### 1.1 打开终端

- 按 `Command + 空格`，输入「终端」或「Terminal」，回车打开。

### 1.2 进入项目目录

在终端里复制粘贴这一行命令，然后按回车：

```bash
cd "/Users/bm/Documents/BitMart Work/预测市场海报模版"
```

> 提示：路径需要和实际存放位置一致。如果你不确定，问一下技术同学把这行命令发给你。

### 1.3 启动服务

```bash
npm start
```

看到这个提示说明启动成功：

```
🎨 海报生成器已启动：http://localhost:3002
```

> 终端窗口保持开着，不要关闭。服务运行期间不需要再动它。

### 1.4 打开生成器页面

浏览器访问：**http://localhost:3002**

收藏这个地址，下次直接从书签打开。

---

## 第二步：填飞书表格

在生成海报之前，先在飞书里把数据填好并保存。各模板的表格链接和填写说明见第 4 节。

---

## 第三步：选模板 → 生成 → 取图

1. 打开 **http://localhost:3002**
2. 点击要生成的模板卡片（如「NBA 赛事」）
3. 点击「**生成海报**」按钮
4. 等待日志滚动完毕，出现「✅ 生成完成！」
5. 点击「**打开输出文件夹**」，直接拿图

当前出图统一规格（所有模板）：

- 格式：PNG
- 尺寸：`900 × 900`
- 体积：单张约 300KB 内

输出目录结构示例：

```
output/
  足球赛事/
    20260417/
      足球赛事_20260417.zip
      poster_01.png
      poster_02.png
```

---

## 第四步：关服务（今天不用了再关）

回到终端，按 `Control + C` 停止服务。下次要用再重新 `npm start`。

## 5. 各模板填写说明

### 5.1 NBA 赛事

必填列：

- `lang` / `title` / `subtitle` / `footer`
- `match1_link` / `match2_link` / `match3_link`（Polymarket 链接）

可选列（不填系统会自动从链接抓取）：

- `match1_home` / `match1_away` / `match1_date`
- `match1_home_win` / `match1_away_win`
- 以上同理适用于 match2、match3

系统自动完成：自动解析比赛主客队、日期和赔率，自动多语言翻译并回填。

---

### 5.2 综合事件

第 2 行主要字段：

- `main title` / `sub title` / `footer`
- `1` / `percent_1`
- `2` / `percent_2`
- `3` / `percent_3`
- 可选 `4` / `percent_4`

系统自动完成：翻译所有文案并写回第 3 行起各语言行。

---

### 5.3 世界杯

基础字段与综合事件一致。配置区（A20:B35）额外支持：

- `scenario_type`（填 `worldcup_winner` 或 `group_winner`）
- `market_slug` 或 `market_url`
- `card_count`

系统自动完成：可从 Polymarket 自动生成卡片内容和概率。

---

### 5.4 足球赛事

第 2 行主要字段：

- `title` / `subtitle` / `footer`
- `match1_home` / `match1_away` / `match1_date`（必填）
- `match2_*` / `match3_*`（可空）
- `match1_link` / `match2_link` / `match3_link`（**强烈推荐填**，赔率最准）

赔率抓取规则：优先按链接抓取，链接无效时按主客队+日期匹配，抓不到时保留原值不阻断出图。

---

### 5.5 币价预测

A 列填语言代码，源语言通常是 `zh-CN`。

主要字段：`main title` / `sub title` / `footer` / `token_1` / `1` / `percent_1`（token_2、3 同理）

系统自动完成：只翻译 title/subtitle/footer，token 名和目标价格保持原值。

---

### 5.6 全球预测市场

最多 3 条卡片，每条包含：`title` / `stat` / `desc`

系统自动完成：翻译 `title` 和 `desc`，`stat` 原样保留。

---

### 5.7 F1 车队

第 2 行主要字段：`main title` / `sub title` / `footer` / `team_1` / `percent_1`（team_2、3 同理）

可选配置（A 列 key，B 列 value）：`market_slug` 或 `polymarket_url`

系统自动完成：标题翻译回填，车队名来自 `teams_f1.csv` 不走翻译。

---

### 5.8 F1 车手

第 2 行主要字段：`main title` / `sub title` / `footer` / `driver_1` / `percent_1`（driver_2、3、4 同理）

可选配置：`market_slug` 或 `polymarket_url`

系统自动完成：标题翻译回填，车手名来自 `drivers_f1.csv` 不走翻译。

## 6. 常见问题

**Q：页面打不开（http://localhost:3002 无法访问）**
服务没有启动。按照第一步重新 `npm start`。

**Q：图里赔率没更新**
先确认飞书表里 `matchN_link` 是否填了有效的 Polymarket 链接；足球赛事推荐优先填链接，准确率最高。

**Q：输出文件夹在哪**
生成完成后点页面上的「打开输出文件夹」按钮会自动打开。手动找的话在项目目录下 `output/模板名/当天日期/`。

**Q：日志里出现红色报错**
截图发给技术同学，附上你选的模板名和飞书表当前内容。

## 7. 更新代码 SOP（技术同学推送更新后）

1. 打开终端，进入项目目录：
   ```bash
   cd "/Users/bm/Documents/BitMart Work/预测市场海报模版"
   ```
2. 拉取最新代码：
   ```bash
   git pull
   ```
3. 如技术同学有说依赖变更，再执行：
   ```bash
   npm install
   ```
4. 重新 `npm start`，按原流程继续出图。

正常情况下不需要重新下载仓库。只有本地目录损坏或换新电脑时才需要重新 clone。
